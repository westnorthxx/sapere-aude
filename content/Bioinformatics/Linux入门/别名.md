### 例子与概念理解
`ll` 命令无法被[[文件查找-which]] 识别，因为 `ll` 是一个别名，并没有被写入 `$PATH` 中

可以用 `alias ll` 查看
```bash
zhoux@westnorth:~$ alias ll
alias ll='ls -alF'
zhoux@westnorth:~$
```
可见 `ll` = `ls -alF`，即以人类可读的方法，包括文件类型，列表形式给出长信息

alias 在 obsidian 中也有应用，很好理解
### 别名时限与永久别名
更换终端或重新登陆终端后，新设定的 `alias` 命令将会被覆盖

将新设定的 `alias` 命令写入配置文件 `~/.bashrc` 可以达成让别名永久生效的效果
```bash
echo "alias rm='rm -i'" >> ~/.bashrc
source ~/.bashrc
```
> [!warning]
> 一定注意**重定向**要使用 `>>` 在配置文件后追加内容，如果使用 `>` 将*覆盖配置文件*，这就完蛋了
